Primary Python File Flow

The main path follows the files doing the most work. Data files, frontend files, tests, and optional services appear as spokes off the file that uses them.

Primary Python flow Read-only JSON/data Frontend Optional external service Support/tests

API entry point

backend/main.py

Defines FastAPI routes, receives frontend requests, calls chatbot, calculation, formatter, and TTS modules.

Conversation brain

backend/chatbot.py

Owns question flow, parsing, branch logic, confirmation, correction handling, and optional LLM use.

Engineering core

backend/wind_load_engine.py

Runs deterministic ASCE 7-16 Chapter 27 calculations. No LLM arithmetic happens here.

Result translation

backend/report_formatter.py

Converts raw engine output into frontend display data and a readable markdown report.

Optional audio

backend/tts.py

Turns spoken_text into audio when TTS is enabled. Fails gracefully when disabled.

How to narrate this diagram The app's centerline is Python: request routing, conversation control, deterministic calculation, and report formatting. JSON files and services are supporting spokes, not decision-makers. The LLM can help with language, but the Python files own trusted state and engineering output.
